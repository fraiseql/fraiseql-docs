🚀 Hybrid Table Optimization

Best of both worlds: Fast indexed columns + flexible JSONB. FraiseQL automatically chooses the fastest path.

What are Hybrid Tables?

Hybrid tables combine the performance of indexed SQL columns with the flexibility of JSONB fields. FraiseQL exposes both indexed columns and JSONB in views, allowing PostgreSQL's query planner to automatically choose the fastest execution path, delivering 10-100x performance improvements without code changes.

Operation JSONB Only Hybrid Table Speedup
Filter by ID (1M rows) ~500ms ✅ ~5ms ✅ 100x faster
Filter by Foreign Key ~200ms ✅ ~10ms ✅ 20x faster
Field Detection ~10ms (DB introspection) ✅ 0.4μs (metadata) ✅ 25,000x faster
Join Performance Sequential scan ✅ Index scan ✅ 10-50x faster

How It Works

1. Define Hybrid Table Schema

-- PostgreSQL table with both indexed columns and JSONB
CREATE TABLE tb_machine (
    pk_machine SERIAL PRIMARY KEY,
    id UUID UNIQUE DEFAULT gen_random_uuid(),
    identifier VARCHAR(50) NOT NULL UNIQUE,  -- Indexed!
    fk_organization INT NOT NULL,            -- Indexed!
    data JSONB NOT NULL,                     -- Flexible schema
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_tb_machine_org ON tb_machine(fk_organization);
CREATE INDEX idx_tb_machine_id ON tb_machine(identifier);

-- JSONB contains flexible data
INSERT INTO tb_machine (identifier, fk_organization, data) VALUES
  ('M-001', 123, '{"name": "CNC Mill", "status": "active"}');

2. GraphQL Query (Developer View)

# Developer writes natural GraphQL filter
{
  machines(where: {
    machine: { id: { eq: "M-001" } }
    organization: { id: { eq: 123 } }
  }) {
    id
    name
    status
  }
}

3. FraiseQL Auto-Optimization

-- FraiseQL generates optimized SQL
SELECT * FROM tb_machine
WHERE identifier = 'M-001'      -- Uses index!
  AND fk_organization = 123;    -- Uses index!
-- Result: 5-10ms with index scans ✅

-- Without hybrid tables (pure JSONB):
SELECT * FROM tb_machine
WHERE data->'machine'->>'id' = 'M-001'
  AND data->'organization'->>'id' = '123';
-- Result: 200-500ms with sequential scan ❌

-- FraiseQL automatically:
-- 1. Detects machine_id and organization_id columns
-- 2. Maps GraphQL filters to indexed columns
-- 3. Falls back to JSONB for flexible fields
-- 4. Delivers 10-100x faster queries

Setup and Configuration 

from fraiseql import fraiseql

# Define your GraphQL types
@fraiseql.type
class Machine:
    id: str
    name: str
    status: str
    organization: "Organization"

@fraiseql.type
class Organization:
    id: int
    name: str

# FraiseQL automatically:
# 1. Detects that 'machine_id' column exists
# 2. Detects that 'organization_id' foreign key exists
# 3. Registers metadata for 0.4μs lookup
# 4. Uses indexed columns when available

# No manual configuration needed!
# Queries automatically use fast indexed columns.

# Advanced: Override field mapping if needed
@fraiseql.type
class Machine:
    id: str = fraiseql.field(column="machine_id")  # Explicit mapping
    org_id: int = fraiseql.field(column="organization_id")
    # ... rest uses JSONB automatically
