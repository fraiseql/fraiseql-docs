Array Operators

Basic Array Operations

overlaps Check if arrays overlap
contains Array contains all elements
contained_by Array is contained by another
any_eq Any element equals value
all_eq All elements equal value

Advanced Array Filtering

# Find products with specific tags
{
  products(
    where: {
      tags: { overlaps: ["electronics", "sale"] }
    }
  ) {
    name
    tags
  }
}

{
  products(
    where: {

Full-Text Search with Ranking

12 Full-Text Search Operators

PostgreSQL's powerful full-text search with ranking, highlighting, and advanced query syntax. Supports multiple languages, stemming, and relevance scoring.

Basic Search

@@ - Full-text match
@@* - Ranked match
@@@ - Query match

Ranking & Scoring

ts_rank - Relevance ranking
ts_rank_cd - Cover density ranking
ts_headline - Highlight results

Advanced Queries

& - AND operator
| - OR operator
! - NOT operator
() - Grouping
Full-Text Search Examples
# Search articles with ranking
{
  articles(
    where: {
      content: { fulltext: "machine learning", rank_gt: 0.1 }
    }
    order_by: { rank: desc }
  ) {
    title
    content
    rank
    headline  # Highlighted search terms
  }
}

# Multi-field search
{
  products(
    where: {
      _or: [
        { name: { fulltext: "wireless" } },
        { description: { fulltext: "wireless" } }
      ]
    }
  ) {
    name
    description
  }
}

Regex & Pattern Matching

POSIX Regular Expressions

Full POSIX regex support with case-insensitive matching, multiline mode, and advanced pattern matching capabilities.

Regex Operators

~ - Regex match (case sensitive)
~* - Regex match (case insensitive)
!~ - Regex not match (case sensitive)
!~* - Regex not match (case insensitive)
~~ - LIKE pattern
~~* - ILIKE pattern

Pattern Examples

# Email validation
{
  users(
    where: {
      email: { regex: "^[\\w.-]+@[\\w.-]+\\.\\w+$" }
    }
  ) {
    email
  }
}

# Phone number patterns
{
  contacts(
    where: {
      phone: { regex: "^\\+?1?[-.\\s]?\\(?[0-9]{3}\\)?[-.\\s]?[0-9]{3}[-.\\s]?[0-9]{4}$" }
    }
  ) {
    name
    phone
  }
}

Performance & GIN Indexes

GIN Indexes for Blazing Performance

Generalized Inverted Indexes (GIN) provide lightning-fast searches on arrays, full-text, and JSONB data. Essential for production filtering performance.

Index Types

-- Array indexing
CREATE INDEX idx_tb_product_tags
ON tb_product USING GIN (tags);

-- Full-text search
CREATE INDEX idx_tb_article_content
ON tb_article USING GIN (to_tsvector('english', content));

-- JSONB path indexing
CREATE INDEX idx_tb_user_metadata
ON tb_user USING GIN (metadata);

-- JSONB expression indexing
CREATE INDEX idx_tb_product_specs_weight
ON tb_product USING GIN ((specs->'dimensions'->'weight'));

Performance Tips

  • Array operations: GIN indexes reduce array queries from O(n) to O(1)
  • Full-text search: ts_vector indexes provide sub-second ranked results
  • JSONB queries: Path indexes accelerate nested property access
  • Composite indexes: Combine filters for multi-column performance
  • Partial indexes: Index only relevant subsets of data

Real-World Use Cases

🛒 E-commerce Product Search

# Advanced product filtering
{
  products(
    where: {
      _and: [
        { category: { eq: "electronics" } },
        { tags: { overlaps: ["wireless", "bluetooth"] } },
        { price: { gte: 50, lte: 500 } },
        { name: { fulltext: "noise cancelling" } },
        { specs: { path_exists: "$.battery.life" } },
        { in_stock: { eq: true } }
      ]
    }
    order_by: { rating: desc, price: asc }
  ) {
    name
    price
    tags
    specs
  }
}

📝 Content Management System

    content
    tags
    author
  }
}
