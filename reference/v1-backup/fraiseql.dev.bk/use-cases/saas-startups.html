Multi-Tenant SaaS Pattern

🤔 FAQ: Do I need one database user per tenant?

No. FraiseQL uses session variables with a shared connection pool. Pass tenant_id in context → FraiseQL runs SET LOCAL app.tenant_id → PostgreSQL RLS policies filter automatically. 

# 1. Create repository with tenant context
from fraiseql.db import FraiseQLRepository

repo = FraiseQLRepository(db_pool, context={
    "tenant_id": current_user.organization_id,  # From JWT/session
    "user_id": current_user.id,
})
# FraiseQL automatically runs: SET LOCAL app.tenant_id = '...'

# 2. Define your models
@fraiseql.type
class User:
    id: UUID
    organization_id: UUID
    email: str
    role: str

# 3. Queries use repository with context
@fraiseql.query
async def users(info, limit: int = 50) -> list[User]:
    # RLS automatically filters by tenant - no manual WHERE needed!
    return await info.context.repo.find("v_user", limit=limit)

# 4. PostgreSQL RLS policy does the work
# CREATE POLICY tenant_isolation ON users
#     USING (organization_id = current_setting('app.tenant_id')::UUID);
