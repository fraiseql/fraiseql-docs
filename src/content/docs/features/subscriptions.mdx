FraiseQL supports GraphQL subscriptions for real-time updates, with multiple transport options optimized for different use cases.

## Overview

Subscriptions push data to clients when events occur:

```graphql
subscription {
    orderUpdated(customerId: "123") {
        id
        status
        updatedAt
    }
}
```

When an order changes, connected clients receive the following WebSocket message:

```json
{
    "id": "1",
    "type": "next",
    "payload": {
        "data": {
            "orderUpdated": {
                "id": "order-456",
                "status": "shipped",
                "updatedAt": "2024-01-15T10:30:00Z"
            }
        }
    }
}
```

The `"type": "next"` field is part of the [graphql-ws protocol](https://github.com/enisdenjo/graphql-ws/blob/master/PROTOCOL.md). Each subscription event arrives as a separate `next` message on the WebSocket connection identified by `"id"`.

<Aside type="note">
**Subscriptions vs. NATS**: FraiseQL subscriptions are the *client-facing* real-time layer — they deliver events to browser and mobile clients over WebSocket. [NATS](/features/nats) is the *backend event bus* used for server-to-server communication and internal fan-out. Most deployments use both: NATS moves events between services, and subscriptions deliver them to end users.
</Aside>

## Quick Setup

<Steps>
1. **Enable subscriptions** in `fraiseql.toml`:
   ```toml
   [subscriptions]
   max_subscriptions_per_connection = 10

   [subscriptions.hooks]
   on_connect    = "http://localhost:8001/hooks/ws-connect"
   on_disconnect = "http://localhost:8001/hooks/ws-disconnect"
   on_subscribe  = "http://localhost:8001/hooks/ws-subscribe"
   timeout_ms    = 500
   ```

2. **Define a subscription** in your Python schema:
   ```python
   import fraiseql

   @fraiseql.subscription(
       entity_type="Order",
       operation="UPDATE"
   )
   def order_updated(customer_id: fraiseql.ID | None = None) -> Order:
       """Subscribe to order updates, optionally filtered by customer."""
       pass
   ```

3. **Compile and run** the server:
   ```bash
   fraiseql compile
   fraiseql run
   ```

4. **Connect a client** using `graphql-ws`:
   ```bash
   npm install graphql-ws
   ```

5. **Subscribe** from your frontend (see client examples below).
</Steps>

## Transports

### WebSocket (graphql-ws)

The default transport for browser clients. Uses the modern graphql-ws protocol.

**Configuration:**

```toml
[subscriptions]
max_subscriptions_per_connection = 10
```

**Client examples:**

<Tabs syncKey="language">
<TabItem label="TypeScript">
```typescript
import { createClient } from 'graphql-ws';

const client = createClient({
    url: 'wss://api.example.com/ws',
    connectionParams: {
        authToken: 'Bearer eyJ...'
    }
});

// Subscribe
const unsubscribe = client.subscribe<{ orderUpdated: { id: string; status: string } }>(
    {
        query: `subscription OrderUpdates($customerId: ID!) {
            orderUpdated(customerId: $customerId) {
                id
                status
                updatedAt
            }
        }`,
        variables: { customerId: '123' }
    },
    {
        next: ({ data }) => {
            console.log('Order updated:', data?.orderUpdated);
            // Output: Order updated: { id: 'order-456', status: 'shipped', updatedAt: '...' }
        },
        error: (err) => console.error('Subscription error:', err),
        complete: () => console.log('Subscription complete')
    }
);

// Clean up when done
unsubscribe();
```
</TabItem>
<TabItem label="Python">
```python
import asyncio
from gql import Client, gql
from gql.transport.websockets import WebsocketsTransport

async def watch_orders(customer_id: str) -> None:
    transport = WebsocketsTransport(
        url="wss://api.example.com/ws",
        headers={"Authorization": "Bearer eyJ..."}
    )

    async with Client(transport=transport) as session:
        subscription = gql("""
            subscription OrderUpdates($customerId: ID!) {
                orderUpdated(customerId: $customerId) {
                    id
                    status
                    updatedAt
                }
            }
        """)

        async for result in session.subscribe(
            subscription,
            variable_values={"customerId": customer_id}
        ):
            order = result["orderUpdated"]
            print(f"Order {order['id']} is now: {order['status']}")
            # Output: Order order-456 is now: shipped

asyncio.run(watch_orders("123"))
```

<TabItem label="Go">
```go
package main

import (
    "context"
    "fmt"
    "log"
    "time"
    
    "github.com/hasura/go-graphql-client"
)

func watchOrders(customerID string) {
    client := graphql.NewSubscriptionClient("wss://api.example.com/ws").
        WithConnectionParams(map[string]interface{}{
            "headers": map[string]string{
                "Authorization": "Bearer eyJ...",
            },
        })
    
    defer client.Close()
    
    subscription := `
        subscription OrderUpdates($customerId: ID!) {
            orderUpdated(customerId: $customerId) {
                id
                status
                updatedAt
            }
        }
    `
    
    variables := map[string]interface{}{
        "customerId": graphql.ID(customerID),
    }
    
    subscriptionId, err := client.Subscribe(subscription, variables, 
        func(data []byte, err error) error {
            if err != nil {
                log.Printf("Subscription error: %v", err)
                return nil
            }
            
            // Parse result
            var result struct {
                OrderUpdated struct {
                    ID        string
                    Status    string
                    UpdatedAt string
                }
            }
            
            if err := json.Unmarshal(data, &result); err != nil {
                return err
            }
            
            fmt.Printf("Order %s is now: %s\n", 
                result.OrderUpdated.ID, 
                result.OrderUpdated.Status)
            return nil
        })
    
    if err != nil {
        log.Fatal(err)
    }
    
    log.Printf("Subscribed with ID: %s", subscriptionId)
    
    // Run for 60 seconds
    time.Sleep(60 * time.Second)
}

func main() {
    watchOrders("123")
}
```

### PostgreSQL LISTEN/NOTIFY

For server-to-server communication without WebSocket overhead.

**Database trigger:**

```sql
-- Automatic trigger generated by FraiseQL
CREATE FUNCTION notify_order_update() RETURNS trigger AS $$
BEGIN
    PERFORM pg_notify(
        'fraiseql_order_updated',
        json_build_object(
            'id', NEW.id,
            'status', NEW.status,
            'customer_id', NEW.fk_customer
        )::text
    );
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

## Defining Subscriptions

### Schema Definition

```python
import fraiseql

@fraiseql.type
class Order:
    id: fraiseql.ID
    status: str
    customer_id: fraiseql.ID
    updated_at: fraiseql.DateTime

@fraiseql.subscription(
    entity_type="Order",
    operation="UPDATE"
)
def order_updated(customer_id: fraiseql.ID | None = None) -> Order:
    """Subscribe to order updates, optionally filtered by customer."""
    pass
```

### Event Filtering

```python
@fraiseql.subscription(
    entity_type="Order",
    operation="UPDATE",
    topic="order_status_changed"  # Narrow to a specific NOTIFY channel
)
def order_status_changed() -> Order:
    pass
```

### Multiple Events

```python
@fraiseql.subscription(
    entity_type="Order"  # No operation filter → receives CREATE, UPDATE, and DELETE events
)
def order_changed() -> Order:
    """Subscribe to any order change."""
    pass
```

## WebSocket Protocol

FraiseQL supports both the modern `graphql-transport-ws` protocol and the legacy `graphql-ws` (Apollo subscriptions-transport-ws) protocol. The protocol is automatically negotiated from the `Sec-WebSocket-Protocol` WebSocket header — no configuration required. Clients without the header default to `graphql-transport-ws`.

### Configuration

```toml
[subscriptions]
max_subscriptions_per_connection = 100  # optional, default: unlimited

[subscriptions.hooks]
on_connect    = "https://your-app/ws/on-connect"     # fail-closed
on_disconnect = "https://your-app/ws/on-disconnect"  # fire-and-forget
on_subscribe  = "https://your-app/ws/on-subscribe"   # fail-closed
timeout_ms    = 500                                   # hook timeout (default: 500ms)
```

### Message Flow

```
Client                          Server
  |                               |
  |------- connection_init ------>|  { type: "connection_init", payload: { authToken: "..." } }
  |<------ connection_ack --------|  { type: "connection_ack" }
  |                               |
  |------- subscribe ------------>|  { id: "1", type: "subscribe", payload: { query: "..." } }
  |<------ next (data) -----------|  { id: "1", type: "next", payload: { data: { ... } } }
  |<------ next (data) -----------|  { id: "1", type: "next", payload: { data: { ... } } }
  |                               |
  |------- complete ------------->|  { id: "1", type: "complete" }
  |                               |
  |<------ ping ------------------|  { type: "ping" }
  |------- pong ----------------->|  { type: "pong" }
```

### Connection Parameters

Pass authentication via `connectionParams`:

```javascript
const client = createClient({
    url: 'wss://api.example.com/ws',
    connectionParams: {
        authToken: 'Bearer eyJ...',
        tenantId: 'tenant-123'
    }
});
```

## WebSocket Reconnection

Network interruptions are common in browser environments. Configure automatic reconnection with exponential backoff:

```typescript
import { createClient } from 'graphql-ws';

const client = createClient({
    url: 'wss://api.example.com/ws',
    connectionParams: { authToken: 'Bearer eyJ...' },

    // Retry indefinitely with exponential backoff (cap at 30s)
    retryAttempts: Infinity,
    shouldRetry: () => true,
    retryWait: async (retries) => {
        const delay = Math.min(1000 * Math.pow(2, retries), 30000);
        await new Promise(r => setTimeout(r, delay));
    },

    // Re-subscribe to active subscriptions after reconnect
    on: {
        connected: () => console.log('WebSocket connected'),
        closed: (event) => console.warn('WebSocket closed:', event),
        error: (err) => console.error('WebSocket error:', err)
    }
});
```

<Aside type="tip">
When a WebSocket reconnects, `graphql-ws` automatically re-sends all active `subscribe` messages. Your application does not need to re-register subscriptions manually — just keep the `unsubscribe` functions from your `client.subscribe()` calls.
</Aside>

**Proxy keepalive**: If your deployment sits behind a load balancer or reverse proxy, configure its idle-connection timeout to be longer than `ping_interval`. For example, with nginx:

```nginx
proxy_read_timeout 90s;   # Must exceed ping_interval (default 30s)
proxy_send_timeout 90s;
```

client.subscribe(

## Error Handling

### Connection Errors

```javascript
const client = createClient({
    url: 'wss://api.example.com/ws',
    retryAttempts: 5,
    retryWait: async (retries) => {
        // Exponential backoff
        await new Promise(r => setTimeout(r, 1000 * Math.pow(2, retries)));
    },
    on: {
        error: (err) => console.error('Connection error:', err),
        closed: () => console.log('Connection closed')
    }
});
```

### Subscription Errors

```javascript
client.subscribe(
    { query: '...' },
    {
        error: (errors) => {
            // GraphQL errors during subscription
            console.error('Subscription errors:', errors);
        }
    }
);
```

## Security

### Authentication

WebSocket connections are authenticated via the same JWT/API key middleware as HTTP requests. The token is passed in the `connectionParams` of the `connection_init` message:

```javascript
const client = createClient({
    url: 'wss://api.example.com/ws',
    connectionParams: {
        authToken: 'Bearer eyJ...'
    }
});
```

The server validates the token before accepting the connection. Rate limiting uses the same `[security.rate_limiting]` configuration as the GraphQL HTTP endpoint.

### Subscription Limits

Limit the number of concurrent subscriptions per connection:

```toml
[subscriptions]
max_subscriptions_per_connection = 10  # optional, default: unlimited
```

## Best Practices

### Use Specific Subscriptions

```graphql
# Good - filtered
subscription {
    orderUpdated(customerId: "123") { id status }
}

# Avoid - unfiltered (receives all orders)
subscription {
    orderUpdated { id status }
}
```

### Clean Up Subscriptions

```javascript
// Store unsubscribe function
const unsubscribe = client.subscribe(...);

// Clean up on unmount
useEffect(() => {
    return () => unsubscribe();
}, []);
```

<Aside type="tip">
Always clean up subscriptions when components unmount or when they are no longer needed. Forgotten subscriptions consume server resources and can degrade performance for all users.
</Aside>

## Troubleshooting

### Connection Drops

1. Check `ping_interval` matches client expectations
2. Verify no proxy timeouts (configure proxy keepalive)
3. Check server logs for connection errors

### Missing Events

1. Verify subscription filter matches data
2. Check auth filter isn't too restrictive
3. Confirm database triggers are active

### High Latency

1. Enable batching for high-frequency events
2. Consider Kafka for extreme throughput
3. Check database NOTIFY backpressure

## Verify It Works

<Steps>

1. Send a subscription over WebSocket:
   ```bash
   # Using wscat (npm install -g wscat)
   wscat -c ws://localhost:8080/graphql -s graphql-ws
   ```
   Then send:
   ```json
   {"type":"connection_init","payload":{}}
   {"id":"1","type":"subscribe","payload":{"query":"subscription { postCreated { id title author { name } } }"}}
   ```

2. In another terminal, create a post to trigger the subscription:
   ```bash
   curl -s http://localhost:8080/graphql \
     -H "Content-Type: application/json" \
     -d '{"query":"mutation { createPost(input: { title: \"Test\", content: \"Hello\" }) { id } }"}'
   ```

3. Your WebSocket connection should receive:
   ```json
   {
     "id": "1",
     "type": "next",
     "payload": {
       "data": {
         "postCreated": {
           "id": "post-abc123",
           "title": "Test",
           "author": { "name": "Alice" }
         }
       }
     }
   }
   ```

</Steps>

<Aside type="tip">If you see the `"type": "next"` message with your post data, subscriptions are working correctly.</Aside>
