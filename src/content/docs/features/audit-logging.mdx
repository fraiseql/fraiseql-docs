FraiseQL provides comprehensive audit logging for compliance and security monitoring, with multiple backend options.

## Overview

Audit logging tracks:
- GraphQL queries and mutations
- Authentication events (login, logout, token refresh)
- Authorization decisions (access granted/denied)
- Administrative operations
- Data access patterns

## Backends

### PostgreSQL

Store audit logs in the database for querying and retention:

```toml
[audit]
enabled = true
backend = "postgres"

[audit.postgres]
table_name = "tb_audit_log"
retention_days = 365
batch_size = 100
flush_interval_ms = 1000
```

### Syslog

Send audit events to syslog for centralized logging:

```toml
[audit]
enabled = true
backend = "syslog"

[audit.syslog]
# RFC 3164 or RFC 5424
format = "rfc5424"

# UDP or TCP
protocol = "udp"
host = "syslog.example.com"
port = 514

# Syslog facility
facility = "local0"

# App name in syslog
app_name = "fraiseql"
```

### File

Write audit logs to files with rotation:

```toml
[audit]
enabled = true
backend = "file"

[audit.file]
path = "/var/log/fraiseql/audit.log"
format = "json"  # or "text"

# Rotation settings
max_size_mb = 100
max_files = 10
compress = true
```

### Multiple Backends

Send to multiple destinations:

```toml
[audit]
enabled = true
backends = ["postgres", "syslog"]

[audit.postgres]
table_name = "tb_audit_log"

[audit.syslog]
host = "syslog.example.com"
```

## Event Types

### Authentication Events

| Event | Description |
|-------|-------------|
| `auth.login` | User login attempt |
| `auth.login.success` | Successful login |
| `auth.login.failure` | Failed login |
| `auth.logout` | User logout |
| `auth.token.refresh` | Token refreshed |
| `auth.token.expired` | Token expired |
| `auth.session.created` | Session created |
| `auth.session.destroyed` | Session ended |

### Authorization Events

| Event | Description |
|-------|-------------|
| `authz.access.granted` | Access allowed |
| `authz.access.denied` | Access denied |
| `authz.role.required` | Role check performed |
| `authz.scope.required` | Scope check performed |

### Operation Events

| Event | Description |
|-------|-------------|
| `graphql.query` | Query executed |
| `graphql.mutation` | Mutation executed |
| `graphql.subscription` | Subscription started |
| `graphql.error` | Operation error |

### Administrative Events

| Event | Description |
|-------|-------------|
| `admin.schema.reload` | Schema reloaded |
| `admin.cache.clear` | Cache cleared |
| `admin.key.rotate` | Encryption key rotated |

## Configuration Options

### Log Levels

```toml
[audit]
# Minimum level to log
level = "info"  # debug, info, warn, error, critical

# Per-event-type levels
[audit.levels]
"auth.login.success" = "info"
"auth.login.failure" = "warn"
"authz.access.denied" = "warn"
"graphql.mutation" = "info"
"graphql.query" = "debug"  # High volume, debug only
```

### Filtering

```toml
[audit.filter]
# Only log specific event types
include = ["auth.*", "authz.access.denied", "graphql.mutation"]

# Exclude specific events
exclude = ["graphql.query"]

# Only log for specific users
include_users = []  # Empty = all users

# Exclude specific users (e.g., service accounts)
exclude_users = ["service-account-id"]
```

### Field Redaction

Protect sensitive data in logs:

```toml
[audit.redaction]
enabled = true

# Fields to redact
fields = ["password", "token", "secret", "ssn", "credit_card"]

# Redaction pattern
pattern = "[REDACTED]"

# Hash instead of redact (for correlation)
hash_fields = ["email", "ip_address"]
hash_algorithm = "sha256"
hash_salt = "${AUDIT_HASH_SALT}"
```

## Querying Audit Logs

### PostgreSQL Queries

```sql
-- Recent failed logins
SELECT * FROM tb_audit_log
WHERE event_type = 'auth.login.failure'
  AND timestamp > NOW() - INTERVAL '1 hour'
ORDER BY timestamp DESC;

-- User activity
SELECT event_type, COUNT(*), MAX(timestamp)
FROM tb_audit_log
WHERE user_id = 'user-uuid-here'
  AND timestamp > NOW() - INTERVAL '7 days'
GROUP BY event_type;

-- Access denied events
SELECT * FROM tb_audit_log
WHERE event_type = 'authz.access.denied'
ORDER BY timestamp DESC
LIMIT 100;

-- Mutations by resource
SELECT
    resource,
    COUNT(*) as mutation_count,
    COUNT(DISTINCT user_id) as unique_users
FROM tb_audit_log
WHERE event_type = 'graphql.mutation'
  AND timestamp > NOW() - INTERVAL '24 hours'
GROUP BY resource
ORDER BY mutation_count DESC;
```

## Retention and Cleanup

### Automatic Cleanup

```toml
[audit.retention]
enabled = true
days = 365
cleanup_interval_hours = 24
batch_size = 10000
```

### Manual Cleanup

```bash
# CLI command
fraiseql-cli audit cleanup --older-than 365d

# SQL
DELETE FROM tb_audit_log
WHERE timestamp < NOW() - INTERVAL '365 days';
```

### Archive Before Delete

```toml
[audit.retention]
archive_before_delete = true
archive_path = "/var/archive/fraiseql/audit"
archive_format = "parquet"  # or "json", "csv"
```

## Best Practices

### Log Mutations, Sample Queries

```toml
[audit.filter]
# Always log mutations
include = ["graphql.mutation", "auth.*", "authz.access.denied"]

# Sample queries (1%)
[audit.sampling]
"graphql.query" = 0.01
```

### Use Structured Details

```python
# Good - structured, queryable
audit.log("graphql.mutation", details={
    "operation": "createUser",
    "input": {"email": "user@example.com"},
    "result": {"id": "user-123"}
})

# Bad - unstructured
audit.log("graphql.mutation", details="Created user user@example.com")
```

### Monitor Audit System Health

```
# Alert on audit write failures
rate(fraiseql_audit_errors_total[5m]) > 0

# Alert on queue backup
fraiseql_audit_queue_size > 1000
```

## Troubleshooting

### Missing Audit Events

1. Check `enabled = true`
2. Verify log level isn't filtering
3. Check filter rules
4. Review backend connectivity

### High Latency

1. Increase `batch_size`
2. Increase `flush_interval_ms`
3. Check database/syslog performance
4. Consider async writes

### Disk Space Issues

1. Enable retention cleanup
2. Reduce retention period
3. Archive to cheaper storage
4. Increase sampling for high-volume events
