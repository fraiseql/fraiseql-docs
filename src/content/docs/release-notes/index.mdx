minor releases may add features and deprecate APIs, patch releases are bugfix
and performance work only. Each row below links to the per-minor page.

## How to use this section

- Pinned at a known version and considering an upgrade? Open the release-notes
  page for the next minor and read the breaking-changes table.
- Want a feature-by-feature view of what was added when? The headline column
  in the table above is the index.
- Migrating between two minor versions across multiple jumps? Follow the
  chain: v2.0 → v2.1, v2.1 → v2.2, v2.2 → v2.3. The framework does not
  publish skip-version migration guides.

## Upgrade guides

The per-version pages cover *what shipped*. For *how to upgrade your code*,
see the upgrading guides:

- [v2.1 → v2.2](/migrations/upgrading/v2-1-to-v2-2/) — mutation response format consolidation and Apollo Federation 2 directive set.
- [v2.2 → v2.3](/migrations/upgrading/v2-2-to-v2-3/) — error-taxonomy consolidation, `ViewName` newtype, and ~16 breaking changes in the Rust core.

For migrating *from another GraphQL or REST framework* (Apollo, Hasura, Prisma,
PostgREST), see [Migrations](/building/migrations/).

## Full history

For releases prior to v2.0 (the v0.x and v1.x series), see the framework
[GitHub Releases](https://github.com/fraiseql/fraiseql/releases) page. Those
versions are not covered by these release notes.
